# 1. Executive Summary

StudyTrack is a frontend web application designed to help self-paced learners organize online courses, track their learning progress and plan study sessions in one place.

Many learners use several platforms simultaneously such as Udemy, Coursera, YouTube, Scrimba, freeCodeCamp and university learning portals — which makes it difficult to stay organized and maintain consistent study habits.

This project aims to provide a simple and focused solution to that problem while also serving as a practical Software Engineering and React portfolio project. It follows the Software Development Life Cycle (SDLC) from planning through deployment, allowing engineering decisions to be documented throughout the development process.

---

# 2. Problem Statement

Self-paced learners often study across multiple online platforms, making it difficult to organize courses, monitor progress and maintain a consistent study schedule.

Existing productivity tools are either too generic, requiring significant customization or too feature-heavy for learners who only need a straightforward way to manage their studies.

As a result, learners may experience:

- Difficulty tracking course progress
- Forgotten study sessions
- Scattered learning resources
- Poor visibility into learning goals
- Reduced motivation due to unclear progress

StudyTrack addresses these challenges by providing a centralized dashboard specifically designed for managing online learning.

---

# 3. Project Goals

The primary goals of this project are to:

- Provide a centralized place to manage online courses.
- Allow users to monitor their learning progress.
- Help users schedule and organize study sessions.
- Display learning statistics through a dashboard.
- Persist user data using browser Local Storage.

This project also aims to strengthen practical skills in:

- Software Engineering
- React development
- Frontend architecture
- Component-based design
- Git and version control
- Project documentation

---

# 4. Business Case

Although this project is being developed as a portfolio application, it addresses a genuine problem experienced by self-paced learners.

The application demonstrates how a focused study planner can simplify course management while applying Software Engineering principles throughout the project lifecycle.

From a learning perspective, the project provides an opportunity to practice:

- Requirements Analysis
- System Design
- UI/UX Design
- Frontend Development
- Testing
- Deployment

in a structured and realistic way.

# 6. Project Scope

## In Scope (MVP)

### Course Management

- Add courses
- View courses
- Edit courses
- Delete courses

### Progress Tracking

- Record completed modules
- Calculate completion percentage
- Display visual progress indicators

### Study Sessions

- Schedule study sessions
- View upcoming sessions
- Delete sessions

### Dashboard

- Total courses
- Overall learning resources
- Upcoming study sessions
- Quick learning summary

### Data Persistence

- Store application data using Local Storage.

## Out of Scope

The following features are intentionally excluded from Version 1:

- User authentication
- Backend services
- Cloud synchronization
- Notifications
- Calendar integrations
- AI recommendations
- Collaboration features
- Mobile application

# 7. Constraints and Assumptions

## Constraints

- Frontend-only application
- Built with React and JavaScript
- Browser Local Storage for persistence
- Developed by a single developer
- Limited development timeline

## Assumptions

- Users access the application using a modern web browser.
- The application is designed for a single user.
- Internet access is only required for deployment and updates.

# 8. Risks

| Risk                              | Impact | Mitigation                            |
| --------------------------------- | :----: | ------------------------------------- |
| Scope creep                       |  High  | Focus on the defined MVP scope        |
| Learning curve                    | Medium | Build features incrementally          |
| Local Storage limitations         |  Low   | Keep stored data lightweight          |

---

# 9. Success Criteria

The project will be considered successful when:

- Users can create, edit and delete courses.
- Learning progress is calculated accurately.
- Study sessions can be managed effectively.
- Data persists after refreshing the browser.
- The application is responsive across common screen sizes.
- The codebase is modular, reusable and well documented.
